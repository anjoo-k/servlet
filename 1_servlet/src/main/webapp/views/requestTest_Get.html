Get방식으로 요청 후 응답페이지 보기

특징1. GET 요청방식으로 요청 시 URL의 Header 영역에 데이터들을 포함시켜 요청 => 사용자가 입력한 값(데이터)들이 url에 노출
=> 보안유지 불가능
=> 즉, 로그인이나 회원가입 같은 경우 GET 방식에 부적합
특징2. Header영역은 전송하는 길이에 제한이 있음 => 방대한 데이터를 담았을 경우 초과하는 데이터는 절단되서 넘어감(잘림) => 즉, 게시판 작성의 경우 GET 방식이 부적합 함(내용물이 너무 길어서)

특징3. url에 데이터가 노출되어 있기 때문에 즐겨찾기 가능(북마크) url을 저장해놨다가 재요청하는 것 => 즉, 검색기능은 GET 방식에 적합
